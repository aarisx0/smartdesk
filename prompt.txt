Build a complete desktop application called SmartDesk AI 
using Electron + React + Node.js + Supabase (PostgreSQL).

SmartDesk AI is an intelligent file organization assistant 
that monitors selected folders, classifies files using IBM 
watsonx Orchestrate, and automatically organizes them with 
user approval. It is NOT a file explorer replacement — it 
works silently in the background as an AI layer on top of 
the OS.

Build src/db/schema.sql (plain SQL to paste 
into Supabase SQL Editor and run once):

CREATE TABLE files (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  filename TEXT NOT NULL,
  extension TEXT,
  filepath TEXT UNIQUE NOT NULL,
  size_bytes BIGINT,
  mime_type TEXT,
  content_preview TEXT,
  status TEXT DEFAULT 'pending' 
    CHECK (status IN 
    ('pending','classified','moved','skipped')),
  suggested_folder TEXT,
  confidence_score FLOAT,
  ai_reasoning TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE user_preferences (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  pattern_keyword TEXT,
  extension TEXT,
  ai_suggested_folder TEXT,
  user_confirmed_folder TEXT,
  times_confirmed INT DEFAULT 1,
  is_learned_rule BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE activity_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  action TEXT CHECK (action IN 
    ('moved','created','deleted','skipped')),
  filename TEXT,
  from_path TEXT,
  to_path TEXT,
  file_size_bytes BIGINT,
  timestamp TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  files_processed INT DEFAULT 0,
  folders_created INT DEFAULT 0,
  duplicates_removed INT DEFAULT 0,
  storage_saved_bytes BIGINT DEFAULT 0,
  session_date DATE DEFAULT CURRENT_DATE
);

CREATE TABLE duplicate_groups (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  file_hash TEXT,
  filenames JSONB,
  sizes JSONB,
  recommended_keep TEXT,
  resolved BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER files_updated_at
BEFORE UPDATE ON files
FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE INDEX ON files(status);
CREATE INDEX ON files(extension);
CREATE INDEX ON activity_log(timestamp);
CREATE INDEX ON user_preferences(pattern_keyword);

Build src/db/queries.js with these functions 
using db.query():
- insertFile(metadata) → INSERT into files, return id
- updateFileStatus(id, status, suggestedFolder, 
  confidence, reasoning) → UPDATE files
- getPendingFiles() → SELECT WHERE status = pending
- getClassifiedFiles() → SELECT WHERE status = classified
- logActivity(action, filename, fromPath, 
  toPath, sizeBytes) → INSERT into activity_log
- savePreference(keyword, ext, aiFolder, userFolder) 
  → INSERT or ON CONFLICT increment times_confirmed
- getLearnedRules() → SELECT WHERE is_learned_rule = true
- getTodaySession() → SELECT or INSERT today's session
- updateSession(id, filesProcessed, foldersCreated, 
  duplicatesRemoved, storageSaved) → UPDATE sessions

Build src/watcher/index.js:
- Watch Desktop and Downloads folders by default
  (configurable via electron-store)
- Ignore: .tmp .crdownload .part files
- On new file detected:
  * Wait 2 seconds (debounce, avoids mid-download)
  * Extract: filename, extension, full path, 
    size in bytes, MIME type using mime-types,
    created/modified date
  * For .pdf: extract first 300 chars using pdf-parse
  * For .txt .md: extract first 300 chars using fs
  * For .exe .msi: extract product name using 
    exiftool-vendored
  * Emit IPC event "file:detected" with full 
    metadata object to Electron main process
  * Insert into Supabase files table 
    with status "pending"

Build src/backend/watsonx.js:
- classifyFile(metadata) async function:
  * Input: { filename, extension, mime_type, 
    size_bytes, content_preview }
  * Check Supabase user_preferences for learned 
    rule matching keyword + extension first
  * If learned rule: return it, confidence 1.0
  * Check Supabase files: if same filename 
    classified before, reuse cached result
  * Otherwise build message:
    "filename=[x], extension=[x], size=[x]MB, 
     mime_type=[x], content_preview=[x]"
  * executeJavaScript on hidden window:
    window.sendToAgent(message)
  * Poll window.classifyFileResult every 500ms
  * Timeout after 15 seconds
  * Strip markdown backticks if present, 
    parse response as JSON
  * Update Supabase: status = classified,
    suggested_folder, confidence_score, ai_reasoning
  * Return: { category, subfolder, 
    suggested_full_path, confidence, reasoning }
  * On failure: return { category: "Unknown", 
    confidence: 0, reasoning: "Service unavailable" }

- Create hidden orchestrate window
- Wait 5 seconds for agent to fully load
- Show "Connecting to SmartDesk AI..." splash screen

Build src/backend/search.js:
- Accept natural language query string
- Parse locally (no AI call needed):
  * Remove stop words: find, where, is, my, the, a
  * Detect file type hint from keywords
  * Remaining words = search keywords
- Search Supabase: ILIKE fuzzy match on filename
  for each keyword, filter by extension if hint found
- If under 3 results: live filesystem search 
  using glob in watched folders
- Rank: exact match first, partial match, 
  then by created_at DESC
- Return: { filename, filepath, size_bytes, 
  extension, updated_at, match_score }
- Record and return search duration
